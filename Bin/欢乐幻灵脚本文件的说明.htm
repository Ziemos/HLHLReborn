欢乐幻灵脚本文件的说明

Bien 2003.10.17

  • 此说明适用与欢乐幻灵Ver0.88版，并与以后版本兼容

  • 脚本文件的格式为xxx.ini，其中xxx自己起名，可以为中文

  • 脚本文件要放到欢乐幻灵目录的Scripts主目录中

  • 脚本中的参数如果为“※用户名”，那么指的就是你在游戏中的名字

脚本文件一共分为如下部分：

1、“[Main]”，脚本起始部分

  • “TransactionCount”，说明要做多少个大步骤（没有限制哟，可以做1000次大步骤，这样......）
    　

  • “RepeatCount”，说明要将脚本重复多少次，如果是“RepeatCount=无限循环”，那么这个脚本就无限循环
    　

2、“[TransactionN]”，描述第N个大步骤，N从0开始

  • “Caption”，说明这个大步骤的标题，会在欢乐幻灵中工作步骤处显示
    　

  • “PurposeCount”，说明这个步骤要达到几个目的，这个目的是为了判断是否需要进行这个大步骤（如果目的中描述的条件在没有进行这个步骤之前就满足了，就不进行这个大步骤），如果不写，就认为没有目的，那么这个大步骤就无条件执行
    注意1：各个目的之间是“与”的关系，即全部的Purpose都满足，才判断为整个目的条件满足
    注意2：Purpose只是在大步骤开始前判断一下，当大步骤开始后，就不再判断了；也就是说，Purpose只是解决“    是否要去做这个大步骤”的问题，而不解决“这个大步骤是否要再做一遍”的问题；如果想在大步骤开始后，等待某一物品的出现，请使用下面要提到的，在Step中的“拥有物品”指令
    　

  • “PurposeN_AllowCount”，描述第N个目的有几种满足条件（其中任意一种满足，就判断为满足，即为“或”的关系），N从0开始
    　

  • “PurposeN_AllowX”,描述第N个目的里面的第X种条件，N和X从0开始
    格式：指令,参数1,参数2,参数3
    目前支持的指令是：
    1、“    物品”，其中

    • 参数1为物品名称

    • 参数2为物品制作者

    • 参数3为应该有几个物品

    • 参数1和2可以有一项空着（但是不要省略逗号），那么就是任意名称或者制作者（最好不要这样）

    2、“宠物”其中

    • 参数1为宠物名称

    • 参数2为宠物应该不低于的忠诚度

    • 参数3为应该有几个宠物

    3、“状态”其中

    • 参数1为目前人物所在状态，有效值为“普通”、“战斗”、“对话”

    4、“地图”其中

    • 参数1为地图ID；如果没有参数2和3，就不对坐标进行要求

    • 参数2为在幻灵地图中的坐标X

    • 参数3为在幻灵地图中的坐标Y

    5、“人物”其中

    • 参数1在目前的有效值是“等级”、“武功数”、“生命百分比”

    • 在参数1为“等级”时：参数2，为人物的称号，有效值为：“凡人”、“散仙”、“地仙”、“夜叉”、“天仙”、“阿修罗”、“大罗金仙”、“魔神”、“天神”、“魔尊”；如果为空，就不进行称号的判断（注意不能省略逗号）
      参数3为人物的等级，格式为xx-yy，指等级在xx到yy的范围中；如果没有-yy，那么就是指等级为xx

    • 在参数1为“武功数”时，参数2可以为具体的数值，或者为“※限制数”，这样就是可以达到的最多武功数

    • 在参数1为“生命百分比”时，参数2为下限，参数3为上限；比如想30%-50%，那么参数2就为30，参数3就为50
      　

  • “StepCount”，说明这个大步骤应该有几个小步骤完成
    　

  • “StepN”，描述第N个小步骤
    格式：指令,参数1,参数2,参数3
    目前支持的指令是：

    • “来到地图”，参数1为要去的地图的ID，可以在map.ini中查到

    • “移动位置”，移动到当前地图某个坐标上，参数1和参数2是要当前地图的坐标

    • “右键单击”，在屏幕上点击右键，参数1和参数2为要点击的相对屏幕坐标，参数3为点击后出现的对话框文字 ，（可以省略，但可能会判断失误）

    • “左键单击”，在屏幕上点击左键，参数和右键单击相同 。如果参数3为“◎战斗状态”，那么就说明点击后需要进入战斗状态；如果参数3为“◎普通状态”，那么就说明点击后需要进入普通状态

    • “点击按钮”，点某个按钮，参数1为这个按钮的窗口名称，参数2为这个按钮父窗口的名称，参数3为点击后出现的窗口的名称

    • “正在战斗”，说明现在应当出于战斗状态，参数1有两个有效值“普通”，“过劫”，分别对应不同战斗情况；参数2如果是“对话”，表明战斗后应当出现对话框，出现的内容就是参数3中描述的内容 ，参数2如果是“普通”，表明战斗后进入普通状态

    • “停止工作”，有的时候，我们会用Purposes来控制脚本的停止，这个命令就是处理这个问题的，参数1是停止工作后欢乐幻灵要显示的提示信息

    • “呼叫NPC”，在可以呼叫NPC的地方呼叫，参数1为NPC的名字
      允许呼叫的NPC的名字：      水城铁匠, 水城药店大夫, 水城饰物店老板, 水城宠物饲养员
       注意：对于“水城铁匠”、“水城药店大夫”和“水城饰物店老板”，必须在某个城中或者是某个药店、饰物店里才可呼叫；对于“水城宠物饲养员”，必须在某个宠物店里面才可呼叫

    • “购买物品”，在幻灵商店中买某个物品，参数1是购买物品的名称，参数2为购买物品的制作者，参数3为购买的数目 ，如果不指定，就买满为止
      注意1：如果身上的该物品数目已经达到了参数3的值，就不会再多购买
      注意2：购买物品后，不会关闭幻灵商店，必须要用      离开商店来关闭幻灵商店；这也方便连续几个Step买在同一个药店不同的物品

    • “离开商店”，离开幻灵商店

    • “进行锻造”，开启五行炉进行炼化，参数1是本次开启五行炉后要做的事情的数据描述节（比如，如果参数1是ForgeN，那么就到[ForgeN]处找描述）

    • “拥有物品”，等待用户身上的某个物品达到指定的数目 （多于或者等于）再继续下一步，参数1是物品的名称，参数2为物品的制作者，参数3为物品需要达到的数目

    • “物品数目”，只有当用户身上的某个物品数目正好为指定数目的时候，再继续下一步，参数1是物品的名称，参数2为物品的制作者，参数3为物品的数目

    • “跳至步骤”，跳转至第N个大步骤，参数1是大步骤号

    • “拥有宠物”，参数1为宠物的名称，参数2为宠物需要达到的忠诚，参数3为宠物需要达到的数目

    • “宠物数目”，参数1为宠物的名称，参数2为宠物需要达到的忠诚，参数3为宠物限定的数目

    • “取消对话”，安全地取消对话，可用于战斗后留在天关

    • “定位窗口”，参数1为窗口名称，参数2为父窗口名名称（如果没有，可以不填）,参数3如果没有，就代表是父窗口地直接子窗口，如果值为“间接”，那么就代表是间接子窗口；注意，这个指令必须在窗口单击和窗口双击的前一个Step才能起作用

    • “窗口单击”，模拟在某个窗口中单击左键，参数1为窗口名称 ，如果为“※”，就使用在定位窗口指令中选择的窗口，参数2、3为X、Y相对坐标

    • “窗口双击”，模拟在某个窗口中双击左键，参数1为窗口名称， 如果为“※”，就使用在定位窗口指令中选择的窗口，参数2、3为X、Y相对坐标

    • “等待延时”，延时一段时间，参数1是毫秒数

    • “捉宠设置”，参数1为要捉的宠物名称，参数2为要捉的宠物的最低级别，参数3为要捉的宠物的最高级别

    • “进入战斗”，参数1为人物动作（攻击、防御、捉宠），参数2为宠物动作（攻击、防御），参数3为如果捉宠没有找到目标时的动作（战斗、逃跑）；注意，这个指令只是使进入战斗，但并不实际战斗，需要用正在战斗指令进行配合

    • “窗口右键”，模拟在某个窗口中单击右键，参数1为窗口名称 ，如果为“※”，就使用在定位窗口指令中选择的窗口，参数2、3为X、Y相对坐标

    • “等待窗口”，一直等待到指定的窗口出现；参数1为窗口名称，参数2为父窗口名名称（如果没有，可以不填）,参数3如果没有，就代表是父窗口地直接子窗口，如果值为“间接”，那么就代表是间接子窗口

    • “开物品栏”，打开物品栏窗口

    • “关物品栏”，关闭物品栏窗口

    • “使用物品”，使用某个物品，参数1是物品的名称，参数2为物品的制作者

    • “武功参数”，设置武功参数，参数1为武功名称，参数2为创招系数，参数3为使用内力

    • “武功外观”，设置武功外观，参数1为起手姿势，参数2为飞行轨迹，参数3为爆炸方式

    • “自创武功”，开始自创武功（一次只创一招）

    • “删除武功”，开始删除武功，参数1为从第几招删，参数2可以为空，也可以为“保留容易”
      　

3、[ForgeN]，表明开启一次五行炉后所要做的事情描述

  • “StoveCount”，表明此次开启五行炉后，要炼化多少次

  • “StoveN”，此次五行炉中第N次炼化的数据描述节 ，N从0开始（比如，如果值是ForgeN_StoveN，那么就到[ForgeN_StoveN]处找描述）


4、[ForgeN_StoveN]，表明一次炼化的描述（开启一次五行炉，可以炼化多次，这里描述一次炼化 过程）

  • “MainRoomPos”，表明主炼物品的位置（位置代码见下面的数据说明）

  • “RoomN_AllowedStuffTypeCount”，第N个位置允许放入的物品种类数，N从0开始（值如果为0，就说明不能放物品）

  • “RoomN_AllowedStuffTypeX_Name”，第N个位置允许放入的第X种物品的名字，X从0开始

  • “RoomN_AllowedStuffTypeX_Attr”，第N个位置允许放入的第X种物品的物品属性码（物品属性码见下面的数据说明）

  • “RoomN_AllowedStuffTypeX_Maker”，第N个位置允许放入的第X种物品的制作者

    大家可以看出来，这个UniverseStove.ini完全避免了炼化物品错误的情况，不会把极限练成飞镖的，嘿嘿

　

下面结合例子，详细解释，红色的字体是说明

; ----------------------- 以下是数据说明 -----------------------
; 物品属性：0-剑, 1-刀, 2-锤, 3-腿, 4-环, 5-扇, 6-杖, 7-叉, 8-钩
; 100-护甲, 200-鞋子, 300-身饰, 400-头饰, 500-暗器, 600-毒药, 700-疗伤药(内药和血药)

; 五行炉中物品位置：0-'乾・金', 1-'兑・金', 2-'离・火', 3-'震・木'
; 4-'巽・木', 5-'坎・水', 6-'艮・土', 7-'坤・土'

; 允许呼叫的NPC的名字：水城铁匠, 水城药店大夫, 水城饰物店老板, 水城宠物饲养员
; 对于“水城铁匠”、“水城药店大夫”和“水城饰物店老板”，必须在某个城中或者是某个药店、饰物店里才可呼叫
; 对于“水城宠物饲养员”，必须在某个宠物店里面才可呼叫

; ----------------------- 以下是行动脚本 -----------------------
[Main]
TransactionCount=3    说明一共有3个大步骤
RepeatCount=2    说明整个脚本循环2次

[Transaction0]    第0个大步骤
Caption=买水城药店的2个无忧失心草＋1个蒙汗药    第0个大步骤的标题
PurposeCount=2    一共有2个目的 ，这两个目的如果都达到，就不进行这个大步骤
Purpose0_AllowCount=1    第0个目的可以有1种允许情况
Purpose0_Allow0=物品,无忧失心草,天晴无名氏,2    目的是 ：有物品，2个天晴无名氏做的无忧失心草
Purpose1_AllowCount=1
Purpose1_Allow0=物品,蒙汗药,天晴无名氏,1    这个就是为了说明一下，其实这个脚本中不用蒙汗药进行锻造
StepCount=4    这个大步骤一共有5个小步骤
Step0=呼叫NPC,水城药店大夫    呼叫水城药店大夫   
Step1=购买物品,无忧失心草,天晴无名氏,2
Step2=购买物品,蒙汗药,天晴无名氏,1    蒙汗药是买着玩的
Step3=离开商店    离开幻灵商店

[Transaction1]
Caption=买水城进行锻造店的8个攒心镖
PurposeCount=1
Purpose0_AllowCount=1
Purpose0_Allow0=物品,攒心镖,天晴无名氏,8
StepCount=4
Step0=呼叫NPC,水城铁匠
Step1=点击按钮,确定(&O),锻造店 - 物品分类,幻灵商店    点击在“锻造店 - 物品分类”中的“确定(&O)”按钮，点击后要出现“幻灵商店”窗口
Step2=购买物品,攒心镖,天晴无名氏,8
Step3=离开商店

[Transaction2]
Caption=进行锻造毒药
StepCount=1
Step0=进行锻造,Forge0    打开五行炉，进行[Forge0]处的描述

Stove1=Forge0_Stove0    第1次，还是按照[Forge0_Stove0]处的描述进行（都一样，呵呵）
Stove2=Forge0_Stove0

Room1_AllowedStuffTypeCount=2    位置1允许放入2种物品
Room1_AllowedStuffType0_Name=无忧失心草    位置1允许放入第0种物品的名称为无忧失心草

Room1_AllowedStuffType1_Attr=600
Room1_AllowedStuffType1_Maker=※用户名    位置1允许放入第1种物品的制作者为玩家在游戏中的名称
